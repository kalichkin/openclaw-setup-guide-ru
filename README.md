# Как поставить своего AI-ассистента (OpenClaw)

> **🤖 Для Claude Code:** Этот файл, пошаговая инструкция по установке OpenClaw на VPS. Ты можешь выполнить её автоматически. Попроси пользователя предоставить: (1) SSH-доступ к серверу (IP + ключ или пароль), (2) выбранный метод аутентификации (Anthropic OAuth, OpenAI OAuth или API-ключ), (3) Telegram bot token и Telegram ID. Затем выполняй установку последовательно через SSH: обнови систему, установи OpenClaw, запусти визард настройки, настрой Telegram, настрой базовую безопасность. Шаги 1-3 пользователь должен сделать сам (регистрации и аккаунты).

---

Этот гайд проведет тебя от полного нуля до работающего AI-ассистента, который доступен 24/7 через Telegram с телефона.

**Что получишь в итоге:** AI-ассистент (Claude или GPT), который живет на твоем сервере и доступен через Telegram-бота. Можно писать ему с телефона из любой точки мира. Он умеет работать с файлами, запускать код, выполнять задачи по расписанию и многое другое.

**Вот план:** Поставить Claude Code на свой компьютер, завести пару аккаунтов, арендовать дешевый сервер, а потом пусть Claude Code разберется со всей технической частью. Итого: примерно 15 минут твоей работы плюс немного ожидания.

**Стоимость:** ~$5/мес за сервер + доступ к AI-модели (см. варианты ниже)  
**Уровень:** Никакого технического опыта не нужно. Серьезно.

## Шаг 1. Установить Claude Code

Claude Code, это AI-ассистент, который работает в терминале. Ты будешь использовать его, чтобы автоматически настроить весь сервер. Считай, что ты нанимаешь технаря, который работает прямо из командной строки.

### 1.1. Открой терминал

- **macOS:** Нажми Cmd+Space, набери "Terminal", нажми Enter
- **Windows:** Поставь [Windows Terminal](https://apps.microsoft.com/detail/9N0DX20HK701) из Microsoft Store. Затем установи WSL2: открой PowerShell от админа и запусти `wsl --install`. После перезагрузки у тебя будет Linux-терминал
- **Linux:** Ты и так знаешь :)

Да, это черный экран с текстом. Нет, он не кусается. Ты удивишься, как быстро терминал станет привычным. Представь, что ты пишешь сообщения компьютеру вместо того, чтобы кликать по кнопкам. Ты что-то набираешь, он отвечает. Вот и всё.

### 1.2. Установи Claude Code

Зайди на [claude.ai/download](https://claude.ai/download) и следуй инструкциям для своей системы. Коротко:

```bash
npm install -g @anthropic-ai/claude-code
```

Если у тебя нет `npm`, сначала поставь Node.js с [nodejs.org](https://nodejs.org) (выбери LTS-версию, прокликай установщик).

После установки запусти:

```bash
claude
```

Claude Code попросит залогиниться. Следуй подсказкам. Как только он тебе ответит, всё готово.

> **Ещё нет подписки Claude?** Ничего страшного. Ты настроишь доступ к AI-модели на следующем шаге, и Claude Code заработает. Также можно использовать Claude Code с API-ключом, если хочешь.

## Шаг 2. Получить доступ к AI-модели (выбери один вариант)

Тебе нужен доступ к AI-модели. OpenClaw поддерживает несколько провайдеров и способов аутентификации. **Выбери то, что подходит:**

---

### Вариант А: Anthropic OAuth (подписка Claude Max/Pro) ⭐ Рекомендуем

**Для кого:** Если у тебя уже есть (или ты хочешь оформить) подписка Claude Pro ($20/мес) или Max ($100/мес).

**Как работает:** Ты авторизуешь OpenClaw через свою существующую подписку Claude с помощью setup-token. Отдельный биллинг за API не нужен.

**Стоимость:** Твоя текущая подписка. Дополнительных расходов нет.

**Что нужно:**
1. Подписка Claude Pro или Max на [claude.ai](https://claude.ai)
2. Claude Code CLI на твоем компьютере (ты только что его поставил в Шаге 1!)
3. Запусти `claude setup-token` в терминале. Это сгенерирует токен, который понадобится позже. **Скопируй его и сохрани.**

**⚠️ Про условия использования:** Условия Anthropic относительно использования потребительских подписок в сторонних инструментах менялись. На начало 2026 года Anthropic заявила, что ничего не меняется в том, как пользователи используют свои аккаунты, и аккаунты не будут отключены. Однако если хочешь максимально чистое соответствие условиям, рассмотри Вариант В (API-ключ). Актуальная информация: [docs.openclaw.ai/providers/anthropic](https://docs.openclaw.ai/providers/anthropic).

---

### Вариант Б: OpenAI OAuth (подписка ChatGPT/Codex) ⭐ Тоже рекомендуем

**Для кого:** Если у тебя уже есть (или ты хочешь оформить) подписка ChatGPT Plus ($20/мес) или Pro ($200/мес).

**Как работает:** Ты авторизуешь OpenClaw через свою существующую подписку ChatGPT/Codex по OAuth. Отдельный биллинг за API не нужен.

**Стоимость:** Твоя текущая подписка. Дополнительных расходов нет.

**Что нужно:**
1. Подписка ChatGPT Plus или Pro на [chatgpt.com](https://chatgpt.com)
2. При настройке OpenClaw выбери "OpenAI Codex". Визард проведет через OAuth-авторизацию

**Про условия использования:** OpenAI явно поддерживает использование подписки через OAuth в сторонних инструментах и рабочих процессах типа OpenClaw. Это самый чистый вариант с точки зрения условий использования среди подписочных опций.

---

### Вариант В: API-ключ (Pay-As-You-Go) 💰

**Для кого:** Максимально чистое соответствие условиям использования. Платишь только за то, что используешь, напрямую через API провайдера.

**Провайдеры:** Anthropic API, OpenAI API или другие (Google Gemini и т.д.)

**Как работает:** Ты получаешь API-ключ у провайдера и платишь за каждый использованный токен. Подписка не нужна.

**Стоимость:** По использованию. Для персонального ассистента обычно **$10-50/мес** в зависимости от интенсивности. Может быть значительно дороже подписки при активном использовании (особенно с моделями с большим контекстом, как Claude Opus).

**Что нужно (Anthropic):**
1. Зайди на [console.anthropic.com](https://console.anthropic.com)
2. Зарегистрируйся (или войди)
3. Привяжи карту для оплаты (Settings → Billing)
4. Зайди в Settings → API Keys → Create Key
5. Скопируй ключ (начинается с `sk-ant-...`). **Сохрани его, он показывается один раз!**

**Что нужно (OpenAI):**
1. Зайди на [platform.openai.com](https://platform.openai.com)
2. Зарегистрируйся (или войди)
3. Привяжи карту для оплаты (Settings → Billing)
4. Зайди в API Keys → Create new secret key
5. Скопируй ключ (начинается с `sk-...`). **Сохрани его, он показывается один раз!**

**Почему этот вариант, а не OAuth?** API, это официально поддерживаемый путь для создания продуктов на базе AI-моделей. Никакой двусмысленности в условиях использования. Если планируешь интенсивное или коммерческое использование, это правильный выбор.

## Шаг 3. Создать Telegram-бота

### 3.1. Создать бота

1. Открой Telegram, найди [@BotFather](https://t.me/BotFather)
2. Напиши ему `/newbot`
3. Придумай имя бота (например: "My AI Assistant")
4. Придумай username бота (например: `my_ai_assistant_bot`). Должен заканчиваться на `bot`
5. BotFather выдаст тебе **токен**, длинная строка вида `7123456789:AAH...`. Сохрани его
6. Чтобы бот мог читать сообщения в группах, напиши BotFather: `/setprivacy`, выбери своего бота, выбери `Disable`

### 3.2. Узнай свой Telegram ID

1. Открой Telegram, найди [@userinfobot](https://t.me/userinfobot)
2. Напиши ему что угодно
3. Он ответит твоим числовым ID (например: `123456789`). Сохрани его

Этот ID нужен для whitelist, чтобы только ты мог писать боту.

## Шаг 4. Арендовать сервер на Hetzner

Тебе нужен маленький и дешевый сервер, где будет жить твой AI-ассистент. Hetzner, это надежный европейский хостинг-провайдер с хорошими ценами.

### 4.1. Создать аккаунт

1. Зайди на [hetzner.com/cloud](https://www.hetzner.com/cloud/)
2. Нажми "Sign Up" (или "Get Started")
3. Заполни регистрацию, подтверди email
4. Привяжи карту (или PayPal)
5. Возможно потребуется верификация (паспорт/ID). Hetzner серьезно относится к KYC. Обычно проходит за несколько часов

### 4.2. Создать SSH-ключ (если нет)

SSH-ключ, это безопасный способ подключаться к серверу без пароля. Один раз настраиваешь и забываешь.

**Проверь, есть ли у тебя ключ:**

```bash
ls ~/.ssh/id_ed25519.pub
```

Если файл существует, у тебя уже есть ключ. Пропусти создание.

**Если ключа нет, создай:**

```bash
ssh-keygen -t ed25519 -C "your@email.com"
```

Жми Enter на все вопросы (пустой пароль, ок для начала).

**Скопируй публичный ключ:**

```bash
cat ~/.ssh/id_ed25519.pub
```

Скопируй всю строку, которая начинается с `ssh-ed25519 ...`

### 4.3. Создать сервер

1. В Hetzner Cloud Console нажми **"Create Server"**
2. **Location:** Выбери ближайший регион (Falkenstein или Helsinki для EU; Ashburn для US)
3. **Image:** Ubuntu 24.04
4. **Type:** Shared vCPU → **CX22** (2 vCPU, 4GB RAM), примерно €4.50/мес (~$5). Хватит с запасом
5. **SSH Key:** Нажми "Add SSH Key", вставь свой публичный ключ (из шага 4.2)
6. **Name:** Дай серверу имя (например `ai-assistant`)
7. Нажми **"Create & Buy Now"**

Через 30 секунд сервер будет готов. Запиши его **IP-адрес** (виден в панели).

**На этом ручная работа закончена.** Не подключайся к серверу по SSH сам. Дальше всё сделает Claude Code.

## Шаг 5. Пусть Claude Code сделает остальное

Вот теперь самое интересное. Ты сделал всю подготовку (аккаунты, ключи, сервер). Теперь передаешь дело дальше.

### 5.1. Открой Claude Code

Вернись в терминал и запусти:

```bash
claude
```

### 5.2. Скажи ему, что делать

Вставь примерно такое (подставь свои данные):

```
Прочитай этот гайд: https://github.com/kalichkin/openclaw-setup-guide-ru

Подключись к моему серверу по SSH (root@ТВОЙ_IP_АДРЕС) и установи OpenClaw.

Вот мои данные:
- Метод аутентификации: [Anthropic OAuth / OpenAI OAuth / API-ключ]
- Setup-токен или API-ключ: [твой токен или ключ из Шага 2]
- Telegram bot token: [твой токен из Шага 3.1]
- Мой Telegram ID: [твой ID из Шага 3.2]

Установи OpenClaw, настрой Telegram и настрой базовую безопасность (файрвол, автообновления).
```

### 5.3. Наблюдай за работой

Claude Code сделает следующее:
1. Подключится к серверу по SSH
2. Обновит систему
3. Установит OpenClaw
4. Запустит визард настройки с твоим методом аутентификации
5. Настроит Telegram (токен бота + твой ID в whitelist)
6. Настроит файрвол и автоматические обновления безопасности
7. Проверит, что всё работает

Если что-то пойдет не так, Claude Code покажет ошибку и предложит решение. Можно просто сказать "да, исправь" и он разберется.

### 5.4. Проверь

Когда Claude Code скажет, что всё готово, открой Telegram, найди своего бота и напиши ему что-нибудь. Если всё настроено правильно, он ответит!

## Сравнение методов аутентификации

| | Anthropic OAuth | OpenAI OAuth | API-ключ (любой) |
|---|---|---|---|
| **Стоимость** | Claude Pro $20/мес или Max $100/мес | ChatGPT Plus $20/мес или Pro $200/мес | По использованию ($10-50+/мес типично) |
| **Условия использования** | Меняются (пока разрешено) | Явно поддерживается | Полностью поддерживается, без двусмысленности |
| **Сложность настройки** | Нужен Claude Code CLI для setup-token | OAuth flow в визарде | Вставить ключ |
| **Лучшая модель** | Claude Opus 4.6 | GPT-5.4 | На выбор |
| **Биллинг** | Фиксированная подписка | Фиксированная подписка | По использованию (может скакать) |
| **Prompt caching** | Недоступен | Н/Д | Доступен (экономит деньги) |

**Наша рекомендация:** Начни с OAuth (Вариант А или Б), если у тебя уже есть подписка. Переходи на API-ключ, если нужно чистое соответствие условиям или хочешь prompt caching.

## Про безопасность OpenClaw (для скептиков)

Валидный вопрос, если ставишь open-source проект с доступом к shell.

**Факты:**
- Код полностью открыт: [github.com/openclaw/openclaw](https://github.com/openclaw/openclaw) (MIT лицензия)
- Нет телеметрии, нет "phone home", нет облачной зависимости
- Данные живут только на твоем VPS. Единственный внешний трафик, это API-вызовы к модели (Anthropic/OpenAI), которые ты бы делал в любом случае
- Формальная Threat Model по MITRE ATLAS фреймворку
- Встроенный security audit: `openclaw security audit --deep`
- Активное community (Discord) + security-контакт (trust.openclaw.ai)
- Модель безопасности: один пользователь = один сервер. Не multi-tenant

**По сравнению с «конвенциональными» продуктами:**
- Claude.ai / ChatGPT: твои данные на их серверах, их правила, их retention policy
- OpenClaw: твои данные на твоем сервере, твои правила, ты контролируешь всё

**Выделенный VPS за $5/мес, это и есть уровень безопасности.** Даже если в OpenClaw найдется баг, он затронет только изолированный сервер, на котором нет ничего кроме ассистента.

## Приложение: Ручная установка (без Claude Code)

Если ты предпочитаешь сделать всё самостоятельно, или если Claude Code столкнулся с проблемами, вот полные ручные шаги. Они покрывают то, что Claude Code делает автоматически в Шаге 5.

### А.1. Подключиться к серверу

```bash
ssh root@ТВОЙ_IP_АДРЕС
```

Если спросит "Are you sure you want to continue connecting?", напиши `yes`.

Должен появиться prompt типа `root@ai-assistant:~#`. Ты внутри.

### А.2. Обновить систему

```bash
apt update && apt upgrade -y
```

Подожди пару минут, пока всё обновится.

### А.3. Установить OpenClaw

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

Скрипт сам установит Node.js (если нет) и OpenClaw. Займет 2-3 минуты.

### А.4. Запустить визард настройки

Команда зависит от выбранного метода аутентификации из Шага 2:

**Вариант А, Anthropic OAuth (setup-token):**
```bash
openclaw onboard --install-daemon --auth-choice setup-token
```
Когда попросит, вставь токен, сгенерированный командой `claude setup-token` на твоем компьютере.

**Вариант Б, OpenAI OAuth (подписка Codex):**
```bash
openclaw onboard --install-daemon --auth-choice openai-codex
```
Визард проведет через OAuth-авторизацию.

**Вариант В, API-ключ Anthropic:**
```bash
openclaw onboard --install-daemon --auth-choice apiKey
```
Когда попросит, вставь свой API-ключ из Шага 2.

**Вариант В, API-ключ OpenAI:**
```bash
openclaw onboard --install-daemon --openai-api-key "ТВОЙ_API_КЛЮЧ"
```

Для всех вариантов:
- **Install as service:** Yes (чтобы работал 24/7)
- Остальные настройки можно оставить по умолчанию

### А.5. Проверить, что работает

```bash
openclaw gateway status
```

Должно показать, что gateway запущен и слушает.

### А.6. Настроить Telegram

Открой конфиг-файл:

```bash
nano ~/.openclaw/openclaw.json
```

Найди (или добавь) секцию `channels` и добавь Telegram:

```json
{
  "channels": {
    "telegram": {
      "token": "ВСТАВЬ_ТОКЕН_ОТ_BOTFATHER",
      "allowFrom": ["ВСТАВЬ_СВОЙ_TELEGRAM_ID"]
    }
  }
}
```

**Важно:**
- `token`, строка от BotFather из Шага 3.1
- `allowFrom`, массив строк (!) с твоим Telegram ID из Шага 3.2

Сохрани файл: `Ctrl+O`, Enter, `Ctrl+X`

### А.7. Перезапустить Gateway

```bash
openclaw gateway restart
```

### А.8. Проверить

Открой Telegram, найди своего бота и напиши ему что-нибудь. Если всё настроено правильно, он ответит!

### А.9. Базовая безопасность

Несколько настроек, чтобы сервер был защищен:

**API-ключи в .env (не в конфиге), только для пользователей API-ключей:**

Если ты выбрал Вариант В (API-ключ), перенеси его из конфига:

```bash
nano ~/.openclaw/.env
```

Добавь:

```
ANTHROPIC_API_KEY=sk-ant-ваш-ключ-здесь
```
(или `OPENAI_API_KEY=sk-ваш-ключ-здесь` для OpenAI)

И убери ключ из `openclaw.json` (OpenClaw подхватит из `.env` автоматически).

```bash
chmod 600 ~/.openclaw/.env
```

Для OAuth-пользователей (Варианты А и Б): токены управляются автоматически. Менять `.env` не нужно.

**Файрвол:**

```bash
apt install -y ufw
ufw default deny incoming
ufw default allow outgoing
ufw allow ssh
ufw enable
```

Это закроет все входящие порты кроме SSH. Gateway слушает на localhost по умолчанию, он и так недоступен снаружи.

**Автоматические обновления безопасности:**

```bash
apt install -y unattended-upgrades
dpkg-reconfigure -plow unattended-upgrades
```

Выбери "Yes". Система будет ставить security-патчи автоматически.

## Troubleshooting

**"openclaw: command not found"**
```bash
export PATH="$(npm prefix -g)/bin:$PATH"
echo 'export PATH="$(npm prefix -g)/bin:$PATH"' >> ~/.bashrc
```

**Gateway не стартует**
```bash
openclaw doctor          # диагностика
journalctl -u openclaw-gateway -n 50   # логи
```

**Бот не отвечает в Telegram**
- Проверь, что токен правильный
- Проверь, что твой ID в `allowFrom` (как строка!)
- Проверь `openclaw gateway status`
- Посмотри логи: `journalctl -u openclaw-gateway -f`

**OAuth-токен истек (Anthropic)**
- Запусти `claude setup-token` заново на своем компьютере
- Затем на сервере: `openclaw models auth paste-token --provider anthropic`

**OAuth-токен истек (OpenAI)**
- На сервере: `openclaw models auth login --provider openai-codex`

**Hetzner не верифицирует аккаунт**
Попробуй DigitalOcean ($6/мес) или Oracle Cloud (бесплатно). Установка аналогична.
